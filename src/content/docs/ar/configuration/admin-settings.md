## Custom Metadata

يستخدم Smarter Files أنواع Custom Metadata Types للتكوين العام. يتيح ذلك نشر الإعدادات عبر المؤسسات من خلال مجموعات التغيير أو Metadata API.

## الإعدادات الرئيسية

### قيود الرفع

حدد قواعد الرفع لكل كائن:

- **Object API Name** — الكائن الذي سيتم تطبيق القاعدة عليه.
- **Allowed Extensions** — أنواع الملفات مفصولة بفواصل (مثال: `pdf,docx`).
- **Max File Size (MB)** — الحد الأقصى لحجم الرفع.

### إدارة الفئات (إصدار AppExchange)

تتحكم الفئات في الأدوار التي يمكنها رؤية ملفات معينة. تتم إدارتها من خلال **معالج التكوين** في تطبيق Smarter Files:

1. افتح تطبيق **Smarter Files by Tucario** من App Launcher.
2. انقر على **Manage Document Categories**.
3. حدد أنواع المستندات، وعيّن الأدوار المسموح بها، ثم انشر.

يتم تخزين الفئات كسجلات Custom Metadata Type (`Tucario_Visibility_Rule__mdt`). يحتوي كل سجل على:

| الحقل | الوصف |
|---|---|
| **Category** | معرّف الفئة |
| **Permitted Roles** | أسماء DeveloperNames للأدوار مفصولة بفاصلة منقوطة |
| **Is Active** | ما إذا كانت القاعدة مفعّلة |
| **Description** | وصف خاص بالمسؤول |

:::note
عند إزالة فئة من خلال المعالج، يتم **إلغاء تنشيطها** بدلاً من حذفها. لا تدعم Salesforce Metadata API حذف سجلات Custom Metadata Type من Apex، لذلك يتم تعيين علامة `Is_Active` للسجل إلى `false`. يتم استبعاد الفئات غير النشطة من جميع الاستعلامات ولا تظهر في منتقي الفئات.
:::

## الصلاحيات

يحترم Smarter Files مشاركة Salesforce القياسية وصلاحيات CRUD. لا يتطلب مجموعات صلاحيات إضافية للوظائف الأساسية.

بالنسبة لإصدار AppExchange، يتم تضمين ثلاث مجموعات صلاحيات:

| مجموعة الصلاحيات | الغرض |
|---|---|
| **Tucario Files** | الوصول الأساسي للتطبيق ووحدات التحكم Apex وكائن الربط. مطلوب لجميع المستخدمين. |
| **Tucario - Manage File Categories** | يمنح القدرة على تعيين فئات للملفات. المستخدمون الذين يمتلكون هذه الصلاحية يتجاوزون التصفية القائمة على الفئات ويمكنهم رؤية جميع الملفات. |
| **Tucario - View Private Documents** | يمنح القدرة على عرض الملفات المعلّمة كخاصة من قبل مستخدمين آخرين. |
